## 2024-05-12 — WordPress Skip-Link Adaptations for Tailwind CSS
**Learning:** When recreating WordPress skip-link accessibility patterns in this Tailwind CSS setup, avoid toggling `sr-only` with `focus:not-sr-only` due to positioning conflicts. Tailwind's `sr-only` utility sets `position: absolute`, but `not-sr-only` sets `position: static` natively, which can break layout by bumping down content when the link becomes visible on focus. WordPress specifically overrides this pattern using absolute positioning combined with `clip`, but Tailwind doesn't have an equivalent single toggle.
**Action:** Instead, explicitly handle positioning and visibility on focus using a combination of position, transform, and left coordinates. Use classes like `absolute -translate-y-full focus:translate-y-0 -left-[9999px] focus:left-4 top-4 z-[100000]` to ensure the link visually drops in without disrupting the normal document flow or triggering CLS (Cumulative Layout Shift).
