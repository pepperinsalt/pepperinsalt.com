## 2024-05-18 — Implementing WordPress-style Skip Links with Tailwind
**Learning:** When recreating WordPress skip-link accessibility patterns in this Tailwind CSS setup, toggling `sr-only` with `focus:not-sr-only` causes layout shifts and positioning conflicts that obscure the skip link. Also, standard React apps lack the automatic target elements often found in WP themes.
**Action:** Instead of `sr-only`, use explicit off-screen positioning (`absolute -translate-y-full -left-[9999px]`) and bring it into view on focus (`focus:translate-y-0 focus:left-4`). Additionally, ensure the target element (e.g., `<main id="main-content">`) has `tabIndex={-1}` and `outline: none` so that the skip link successfully shifts browser focus without showing an ugly focus ring around the entire main content area.
