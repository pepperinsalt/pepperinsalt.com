## 2024-05-11 — Skip Link in React SPA with Tailwind
**Learning:** Adapting the classic WordPress skip-link accessibility pattern (`screen-reader-text` class to `#main`) to a React SPA built with Vite and Tailwind CSS. The `screen-reader-text` utility from WP is not available here, so Tailwind utilities `sr-only` and `focus:not-sr-only` must be used. Adding `focus:absolute` and positional attributes like `focus:top-4 focus:left-4 focus:z-[100000]` recreates the typical WordPress interaction behavior when keyboard tabbing begins.
**Action:** Always use Tailwind utility class `sr-only` coupled with `focus:not-sr-only focus:absolute` to recreate WordPress skip links in modern frontend stacks. Ensure the routing component sets up a consistent `<main id="main-content">` wrapper for the anchor destination.
