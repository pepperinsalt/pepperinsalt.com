## 2026-05-13 — [SPA Accessibility: Adapting WordPress Skip Links to React]
**Learning:** When adapting the standard WordPress skip-to-content accessibility pattern to a React SPA built with Tailwind, relying on utility classes like `sr-only` coupled with `focus:not-sr-only` can create positioning conflicts or unintended layout shifts. Moreover, simply adding the link isn't enough; the target (`<main id="main">`) must be given `tabIndex={-1}` to be programmatically focusable without being in the regular tab order, and it must have `outline: none` on focus to prevent an ugly focus ring when the skip link is used.
**Action:** Always implement skip links with explicit absolute positioning (`left: -9999px`) and reset those coordinates on `:focus`. Ensure the target `<main>` tag has `tabIndex={-1}` and `main#main:focus { outline: none; }` to maintain both functionality and aesthetics.
