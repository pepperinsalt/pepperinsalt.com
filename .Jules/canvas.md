## 2024-05-16 — WordPress Skip Link Pattern in React/Tailwind
**Learning:** When recreating the WordPress skip-link accessibility pattern (`screen-reader-text`) in this React application utilizing Tailwind CSS v4, toggling `sr-only` with `focus:not-sr-only` causes layout conflicts and positioning issues. Native WordPress styles handle this with clipping and absolute positioning.
**Action:** In a Tailwind environment, explicitly manage the positioning and transform states instead of relying on `sr-only`. Use utility classes like `absolute -translate-y-full focus:translate-y-0 -left-[9999px] focus:left-4 top-4 z-[100000]` to slide the skip link into view upon focus while keeping it out of the document flow. Ensure the target (e.g. `<main>`) has `tabIndex={-1}` and `outline: none` on focus (`focus:outline-none`) so the user context smoothly transitions without a lingering focus ring.
