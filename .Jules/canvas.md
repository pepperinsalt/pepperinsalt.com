## 2026-05-14 — Adapt WP Skip Link for React/Tailwind
**Learning:** Traditional WordPress skip links often use custom CSS classes like `.screen-reader-text` which do not inherently exist in React applications using Tailwind CSS. When applying `.sr-only` to an element and then toggling it on focus using `.focus:not-sr-only` it causes visual and positioning issues due to how Tailwind's sr-only explicitly removes properties rather than properly moving the element.
**Action:** Instead of relying on `sr-only`, implement the skip-to-content link using Tailwind positioning utilities that explicitly remove it offscreen and translate it into view upon focus (`absolute -translate-y-full focus:translate-y-0 -left-[9999px] focus:left-4 top-4 z-[100000]`). The targeted `<main>` element must have `tabIndex={-1}` and `className="focus:outline-none"` to programmatically receive focus without introducing an unnatural visible focus ring where it shouldn't belong.
