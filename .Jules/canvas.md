## 2026-05-17 — WordPress Accessibility Pattern Adaptation (Skip Link)
**Learning:** When adapting the standard WordPress skip-to-content accessibility pattern to a React/Tailwind codebase, using Tailwind's standard `sr-only` class alongside `focus:not-sr-only` can cause positioning conflicts depending on the surrounding layout, as `not-sr-only` resets properties back to defaults that might not position the skip link correctly at the top of the viewport.
**Action:** Instead of `sr-only`, explicitly manage visibility and positioning on focus with explicit coordinate classes like `absolute -translate-y-full focus:translate-y-0 -left-[9999px] focus:left-4 top-4 z-[100000]`. Ensure the target `<main>` area is focusable but visually clean via `tabIndex={-1}` and `className="outline-none focus:outline-none"`.
